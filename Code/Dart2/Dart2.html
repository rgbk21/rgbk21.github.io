Chapter 1: Hello Dart
Installing the Dart SDK
- Follow the instructions here to install the SDK.
- Verify that the SDK has been installed by running: dart --version.
- You can run dart code on dartpad.dev.
- Whenever you download and open a new Dart project that contains Pub packages, you'll need to run dart pub get in the root folder of your project. This will download any libraries mentioned in pubspec.yaml file. - Check the contents of the SDK using dart help. This will show you the list of tools that make up the SDK. This is a list of what they are and what they do:
a) analyze: Your IDE uses this tool to tell you when you've made a mistake in your code.
b) compile: This tool compiles Dart code into an optimized native executable program for Windows, Linux or macOS. This is known as ahead-of-time, or AOT, compilation. Alongside native executables, web technologies are another major focus for Dart, so you can also use the compile tool to convert Dart code to JavaScript.
c) create: This is for creating new Dart projects.
d) fix: The fix tool is there to help developers update their old projects to use the shiniest new Dart syntax.
e) format: Automatic indentation of your code.
f) migrate: Helps migrate old projects to use null safety.
g) pub: Pub is the name of the package manager for Dart, and pub is the tool that handles the job. A package is a collection of third-party code that you can use in your own Dart project. You can browse packages available to you on pub.dev.
h) run: This runs your Dart program in the Dart Virtual Machine, or VM. The Dart VM compiles your code right before it's needed. In contrast to AOT, this is known as just-in-time, or JIT, compilation, which will let you make small changes to your code and rerun it almost instantly. This is especially useful for applications like Flutter where you'll need to make lots of little changes as you refine the UI.
i) test: Unit testing.
Running Dart
- You can run a single dart file by going to the folder that has the file and then running dart run myDartFile.dart.
- If you want to create a whole project, you can run dart create hello_world_project. This will create a Dart project with some default code. You can run this project by cd hello_world_project, and then run the project by doing dart run bin/hello_dart_project.dart. The run keyword is optional in the statement.
- The above command to create a project would create a simple console app because that is the default. You can create other types of project by using the --template option. You would use it like this: dart create --template console-full hello_dart_project.
Browsing the generated code

For example, in order to pass args to a dart program, you would do: dart myDartAppName args1 args2 args3. If you don't want to pass args, you can remove that from the method definition.

Chapter 2: Expressions, Variables & Constants
Adding comments
- You add documentation comments in dart by using /// for single line comments or /** */ for block comments. Documentation comments also support Markdown formatting.
Statements and Expressions
- A statement is a command, something that you tell your computer to do. A simple statement is something that ends in a semicolon. So print('Hello, Dart Apprentice reader!'); is a statement.
- There are also complex statements and code blocks that use curly braces, but there's no need to add semicolon after the braces.
- Unlike a statement, an experession doesn't do something. In other words, an expression is a value or something that can be calculated as a value. eg. 42, 3+2, 'Hello World'. These are all expressions.
Arithmetic Operations
- Doing something like 22/7 would print the output as 3 in some languages. Not in dart. It will give you the value 3.142857142857143. If you actually did want to perform integer division, you do 22 ~/ 7. The ~/ operator is officially called the truncating division operator when applied to numbers.
- The modulo operator is %. So 28 % 10 will print 8.
- The ~/ and % operators have the same precedence as multiplication and division.

Constants
- Dart has two different types of 'variables' whose values never change. They are declared with the const and final keywords. There is a difference between the two keywords.

a)

const constants

:
- To create a constant in Dart, use the const keyword. const myConstant = 10;. Just like with the var keyword, Dart uses type inference to determine that myConstant is an int.
- Once you've declared a constant, you can't change its data. So myConstant = 0 is not allowed.

b)

final constants

:
- Often, you know you'll want a constant in your program, but you don't know what its value is at compile time. You'll only know the value after the program starts running. This kind of constant is known as a runtime constant.
- In Dart const is only used for compile-time constants; that is, for values that can be determined by the compiler before the program ever starts running.
- If you can't create a const variable because you don't know its value at compile time, then you must use the final keyword to make it a runtime constant. There are many reasons you might not know a value until after your program is running. For example, you might need to fetch a value from the server, or query the device settings, or ask a user to input their age.
- For example, final hoursSinceMidnight = DateTime.now().hour;
- As a general rule, try const first, and if the compiler complains, then make it final. The benefit of using const is it gives the compiler the freedom to make internal optimizations to the code before compiling it.

Strings
- Everything in a computer is denoted as a number. So are Strings. Strings are a sequence of characters. Each character is represented as a unique number in the computer. This two-way mapping between a character and a number is called a character set. In isolation, a computer could use any kind of mapping between a character and a number. For instance, you could use 1 to denote A, 2 for B and so on. But in a world where computers need to share data, you want a certain standard so that when a computer sends a sequence of numbers (that denote a string) to another, that other computer can reproduce the original string.
- There have been different standards over the years, but the most modern one is Unicode, at unicode.org.
- For instance, the word "cafe" would be mapped to the following numbers "99 97 102 101" in unicode. Each of the number associated with a character is known as code point. So we would say that the letter "c" has a code point of 99.
- Unicode maps letters in other languages as well. For instance, " Êàë Áà± ‰Ω† " would be mapped to "25105 29233 20320".
- Similarly, in dart, strings are represented as a sequence of characters as well. But what is different in the case of Dart is that it doesn't have a Character or char type. What this means is that in final firstLetter = 'a';, even though firstLetter is a single character, it is still of type String.
- Even though everything is a String, you can still get the underlying characters.
- Dart strings are a collection of UTF-16 code units. UTF-16 is a way to encode Unicode values by using 16-bit numbers. You can get the underlying character value by using codeUnits.

Objects and Dynamic Types
- JavaScript is a dynamically-typed language. Dynamic means that something can change, and for JavaScript that means the types can change at runtime. Dart is an optionally-typed language. That means you can choose to use Dart as a dynamically typed language or as a statically-typed language. Static means that something cannot change; once you tell Dart what type a variable is, you're not allowed to change it anymore.
- Like we saw earlier, dart has a dynamic type. A variable that has been declared as dynamic can be assigned any type of value.
- Also, if you declare a variable as var and don't assign it a value, dart treats that variable as type dynamic.
- If you need to explicitly say that any type is allowed, you should consider using the Object? type.
- At runtime, Object? and dynamic behave nearly the same. However, when you explicitly declare a variable as Object?, you're telling everyone that you generalized your variable on purpose, and that they'll need to check its type at runtime if they want to do anything specific with it. Using dynamic, on the other hand, is more like saying you don't know what the type is; you're telling people they can do what they like with this variable, but it's completely on them if their code crashes.
Chapter 4: Control Flow
Boolean Operations
- In the case of an else-if ladder, only the code associated with the first true condition encountered will be executed, regardless of whether there are subsequent else-if conditions that evaluate to true. In other words, the order of your conditions matters. You can add an else clause at the end to handle the case where none of the conditions are true. This else clause is optional if you don't need it.
- The ternary conditional operator takes a condition and returns one of two values, depending on whether the condition is true or false. The syntax is as follows: (condition) ? valueIfTrue : valueIfFalse;.
- In Dart, switch statements don't support ranges like number > 5. Only == equality checking is allowed. If your conditions involve ranges, then you should use if statements. switch statements can also work with Strings.

- Name of the enum is normally written with a capital letter, for instance, Weather. The individual enums are written in camelCase.
- You can use for-in loops to iterate over collections. It doesn't provide an index. Example of using enum.

Making parameters optional
- To indicate that a parameter is optional, you surround the parameter with square brackets and add a question mark after the type. Adding the ? would mean that the parameter can be null, in which case the method would have to handle the case where the parameter is null explicitly. Closures and Scopes
- Anonymous functions in Dart act as what are known as closures. The term closure means that the code "closes around" the surrounding scope, and therefore has access to variables and functions defined within that scope.
- A "scope" in Dart is defined by a pair of curly braces. All the code within these braces is a scope. You can have nested scopes within other scopes. Examples of scopes are function bodies and bodies of loops.

Chapter 6: Classes
Introduction to classes
- All values in Dart are objects that are built from a class. This includes the values of basic types like int, double, and bool. That's different from other languages like Java where basic types are primitive. For example, if you have x = 10 in java, the value of x is 10 itself. However, Dart doesn't have primitive types. Even for a simple int, the value is an object that wraps the integer.
Expand Gist

Understanding Object Serialization
- Serialization is the process of converting a complex data object into a string. Once the object has been serialized, it's easy to save that data or transfer it across the network because everything from your app to the network and beyond knows how to deal with strings. Later, when you want to read that data back in, you can do so by way of deserialization, which is simply the process of converting a string back into an object of your datatype.
- Serialization and Deserialization are such common tasks that people have devised a number of standardized formats for serializing data. One of the most common ones is called JSON: JavaScript Object Notation.
- In JSON, objects are surrounded by curly braces, properties are separated by commas, property names are separated from property values by colons, and strings are surrounded by double-quotes. If a string needs to include double-quotes inside itself, you can escape it with a backslash like so: \".
- JSON is very similar to a Dart data type called Map. In fact, Dart even has built-in functions in the dart:convert library to serialize and deserialize JSON maps. We will look at this later.

Factory Constructors
- All the constructors we saw above were generative constructors, which means that they can only create a new instance of the class.
- Factory constructors on the other hand can return existing instances oth the class, or even subclasses of it.
- The official dart docs have a better explanation of it.
- Use the factory keyword when implementing a constructor that doesn't always create a new instance of its class. For example, a factory constructor might return an instance from a cache, or it might return an instance of a subtype. Another use case for factory constructors is initializing a final variable using logic that can't be handled in the initializer list.
