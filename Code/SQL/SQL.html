Chapter 1: Data and Tables - A Place for Everything
- If you have a bunch of data that you want ot organize into a table, the first thing that you need to do is identify the common bits of data and then assign that common bit a column name.
What is a Database
- A database is a structure that holds tables and other SQL structures related to those tables. A table is the structure inside your database that contains data, organized in columns and rows. Each category that you identified earlier becomes a column. All the information about one object in your table becomes a row. You will often see the word field used instead of a column. They mean the same thing. Also, row and record are used interchangeably.
- Databases contain connected data - meaning - all of the tables in a database should be connected in some way.
- The SQL language requires all tables to be inside a database. SO even if you have a single table, you need to place it inside a database. One of the features of SQL is to control access to your tables by multiple users. Being able to grant or deny access to an entire database is sometime simpler than having to control the permissions on each one of the multiple tables.
- SQL does not allow you ot recreate an existing TABLE or DATABASE. If you try to create a new table that has the same name as an existing table, it will give you a "Table already exists Exception". (Even if the new table taht you are creating is different from the existing table). Hence to create a new table that has the same name as an existing table, you will have to first delete the existing table.
Creating a Database
CREATE DATABASE gregs_list;
This will create the database. Spaces are not allowed in the name of the database. Command must end with a semi-colon.

USE gregs_list;
This will tell your RDBMS to actually use the database that we just created. Now everything we do will happen only inside the gregs_list database.

CREATE TABLE my_contacts
(
first_name VARCHAR(30) NOT NULL,
last_name VARCHAR(30) NOT NULL,
email VARCHAR(30),
gender CHAR(1),
birthday DATE,
numOfPets INT NOT NULL DEFAULT 0
);
- The comma separates the columns being created. first_name and last_name.. are the names of the columns. VARCHAR(30) specifies the datatype. It is used to hold information stored as a text. The (30) means that the text it holds can be up to 30 characters long.
- If there are certain columns in your table that should always have values, you can set up your table to not accept NULL values for those columns. For example, in the above table, you MUST provide a value for the first_name, last_name and numOfPets columns in the INSERT statement.
- The value that follows the DEFAULT keyword is automatically added if no other value is specified. The default value has to be of the same type of value as the column. This means that the numOfPets can not only be NOT NULL, but if no value is specified for the column, the value of 0 will be automatically inserted into the table, thus satisfying the requirement of the NOT NULL.
- So the question is: which columns should be assigned to be NOT NULL? Any columns that you might use later to search with, or any unique columns should be NOT NULL. Ideally you want the data in your table to be complete. However, if you know a column that will be filled in later, you can leave it as NULL.
- Remember that a column with a NULL value IS NULL but does not EQUAL NULL.

DESC my_contacts;
This will show what your table looks like. Note that this will show ONLY the structure of your table. Not the information inside of it.

SHOW CREATE TABLE my_contacts;
This will return the CREATE TABLE statement that can exactly recreate our table. The output from this command is also going to contain bacticks and what not. You can paste the output directly into the console and recreate the table. The bacticks and data settings don't really affect it.

Chapter 2: The SELECT Statement - Data Retrieval
- Instead of using the SELECT statement to return all the entries in the database, we can use it with the WHERE clause to fine-tune our query.

SELECT * FROM my_contacts
WHERE first_name = 'Anne';
- Putting first_name with the WHERE clause tells SQL that you want to look in only the column called first_name. So what this query is saying is: return all the columns of the record r from my_contacts table where r.first_name = 'Anne'. Whenever you see *, as in SELECT *, think of it like asking your SQL software to SELECT ALL THE COLUMNS.
- Notice that a query on VARCHAR column datatype should always be inside quotes ''. Hence it's 'Anne' and not Anne. If the datatype is an INTEGER or a DECIMAL, then you do not need quotes. Only the numeric types - INTEGER and DECIMAL do not need quotes. The VARCHAR, CHAR, BLOB, DATE, and TIME datatypes all require quotes.
- You need to escape the single quote - use the \ (backslash) to escape the quote. for eg. WHERE first_name = 'D\'Antonio'.
- Do not use double quotes to specify a VARCHAR in your SQL code. You use " in the programming language to say "this is where the SQL statement is". That way, single quotes are recognized as being part of the SQL syntax and not the programming language.

SELECT email, last_name FROM my_contacts
WHERE first_name = 'Anne';
- Instead of using SELECT * that returned all the columns from the table, here we are specifying the columns that we want to be returned - namely - email and last_name. Note that different column names are spearated by a comma. This returns your queries faster.
Similarly you could have also used WHERE first_name <> 'Anne';
. This would return all reuslts where the first name is NOT EQUAL to 'Anne'.

SELECT email, last_name FROM my_contacts
WHERE first_name = 'Anne'
AND numOfPets = 1;
This is using the AND keyword to combine two WHERE clauses. Note the absence of qotes in numOfPets. And that comparison is done using just a single = sign. Not ==. Similarly, you can replace the equals comparison with greater than AND numOfPets > 0 or NOT EQUAL as in AND numOfPets <> 0.

SELECT email, last_name FROM my_contacts
WHERE first_name => 'A'
AND first_name < 'B';
This query will return contacts whose first letter is A or later, but whose first letters come earlier than the alphabet B.

SELECT email, last_name FROM my_contacts
WHERE first_name = 'Anne'
OR numOfPets = 1;
Similar to AND, you can also combine 2 queries using OR.

SELECT email, last_name FROM my_contacts
WHERE email IS NULL
This is the ONLY way to select a NULL value. By uisng the keyword IS NULL. DO not use things like WHERE email = NULL or WHERE email = 'NULL'. These are all wrong.

SELECT email, last_name FROM my_contacts
WHERE location LIKE '%CA';
A % sign within single quotes tells SQL that we are looking for all values in the location column that END with 'CA'. The % sign is a stand-in for any number of unknown characters. Another wildcard is _. It's a standin for just ONE unknown character. So WHERE location LIKE '_CA' would have returned rows containing locations like Aca, Bca, Cca...
You could also have something like WHERE location LIKE 'I%';. This would have returned all locations starting with the letter I. You could also have WHERE location LIKE '_e%'; which would have returned locations like Texas, Delhi, etc.
Remember that if your column contains complicated information, LIKE may not be specific enough to target precise data.

SELECT email, last_name FROM my_contacts
WHERE numOfPets BETWEEN 1 AND 3;
BETWEEN is equivalent to using the <= and the >= symbols, but not the > and < symbols. You can also use the BETWEEN keyword for text data. SO you can write WHERE first_name BETWEEN A AND B. This will return the names of all the contacts that have the first character as A, or B, and everything in between.
- The order matters in the BETWEEN keyword. So the smaller value should always be on the left and the larger value on the right. So this would be a wrong query: WHERE numOfPets BETWEEN 3 AND 1;

SELECT email, last_name FROM my_contacts
WHERE first_name IN ('Anne', 'Bianca', 'Charlize');
- Instead of using WHERE first_name = 'Anne' OR first_name = 'Bianca' OR first_name = 'Charlize' we can simplify it by using the IN keyword. Use IN with a set of values in parentheses. When the value in the column matches the value in the set, the entire row or the specified columns are returned.
- Similarly, you can use the NOT IN operator WHERE first_name NOT IN ('Anne', 'Bianca', 'Charlize');.

SELECT * FROM my_contacts
WHERE numOfPets NOT BETWEEN 2 AND 4;
- This would return the rows where the numOfPets < 2 OR numOfPets > 4. Notice the lack of the equals sign.

SELECT * FROM my_contacts
WHERE NOT first_name LIKE 'A%'
AND NOT gender LIKE 'M';
- This wll return the contacts where the first_name does not start with an A and the gender is not M.

SELECT * FROM my_contacts
WHERE email IS NOT NULL;
- This wll return the contacts from the table where email is not NULL.
Chapter 3: DELETE and UPDATE - Changes are good
- Information in a table is not stored chronologically. So it's not like the record that occurs at the bottom of the table is the record tha twas added the last. You should not rely on the rows in the table being in chronological order.
- We use the DELETE keyword to remove an entire record from the table. Note that in order to DELETE rows from the table, you will have to turn off the "Safe Updates" by going to Edit - Preferences - SQL Editor - Uncheck the checkbox for safe updates.

DELETE from my_contacts
WHERE gender = 'M';
- Couple of points to note. Unlike the SELECT statement, we do not have to tell it what to delete. It will automatically delete the entire row. Also, make sure that you do not forget the WHERE clause or else it will delete all of the rows in your table. So never run something like DELETE FROM my_contacts;.
- The WHERE clause is the exact same as you used in the SELECT statements. Just like the previous scenarios, you can combine it with LIKE, IN, BETWEEN to tell the RDBMS what rows to exactly delete.

DELETE FROM my_contacts
WHERE email IS NULL;
INSERT INTO my_contacts
(first_name, last_name, email, gender)
VALUES
('Eva', 'Doe', 'edoe@gmail.com', 'F');
- You can use the INSERT-DELETE combo, as shown above, to update the table. First DELETE the row and then INSERT a new row that has the updated details.
- DELETE carefully. Make sure you include a precise WHERE clause to target the exact rows you really want to delete. Unless you are absolutely certain that your WHERE clause will delete the rows you want it to, you should use a SELECT first to make sure. Since DELETE and SELECT can make the same use of the same WHERE clause, the rows that the SELECT returns will echo the rows that the you'll DELETE with that WHERE clause.

UPDATE my_contacts
SET
first_name = 'Evelyin',
email = 'evedoe@gmail.com'
WHERE first_name = 'Eva';
- Instead of inserting a new row and deleting the old one, you can reuse an existing row, changing only the column values that you want to change. This is the UPDATE statement. The SET keyword tells the RDBMS that it needs to change the column before the equal sign to contain the value after the equal sign. The WHERE clause tells RDBMS on which columns the changes should be applied.
- You can change multiple rows in one go by using commas to separate the SET clauses. Also note how we are searching using the first_name in the WHERE clause and updating the same first_name in the SET clause.
- Just like before, if you leave out the WHERE clause, you will end up changing the first_name of EVERY record to Evelyin. So don't do that.

UPDATE my_contacts
SET
numOfPets = numOfPets + 1
WHERE first_name = 'Evelyin';
- In SQL, we can perform basic math operation on number columns. We are incrementing the numOfPets by 1 in the above example.
Chapter 4: Smart Table Design - Need for being Normal
How you are going to use your data will define how you set up your table. For example, if a column containing address is present, you can choose to either keep the entire address as a single column, or divide the columns into separate Street Number, Unit Number, and so on. What you should not do is keep both - one column containing th entire address and the other containing the address split into its constituents. Because if you then have to perform an update operation on hte address, you will have to update both of them. So, should you split your address or not? Well, depends on how you are going to be using your data.
- Remember that SQL is a relational database. This means that to design a table, you need to consider how the columns relate to each other to describe a thing.
- Broad steps that you can follow when defining a table:
  • Pick the one thing you want your table to describe
  • Make a list of the information you need to know about your one thing when you are using the table
  • Using the list, break down the information about your thing into pieces you can use for orgainizing your tables.
What is atomic data
- When a data in a table is atomic, it means that the data has been broken into the smallest piece of data that can't or shouldn't be divided. These are some question you can ask to help you figure out what you need to put in your tables:
  • What is the one thing your table describes
  • How will you use the table to get that one thing
  • Do your columns contain atomic data to make your queries short and tho the point
- Note that making your data atomic means breaking it down into the smallest pieces that you need to create an efficient table; not just the smallest pieces you possibly can. Atomic data alos helps you to ensure that the data in your table is accurate. For example, if you have a separatae column for street numbers, you can ensure that only numbers end up in that column. This also makes the computation of the queries faster.
- There are two official rules of atomic data:
  • A column with atomic data cannot have several values of the same type of data in that column
  • A table with atomic data can't have multiple columns with the same type of data
What is FIRST NORMAL FORM or 1NF
- To be 1NF, a table must follow these rules:
  • Each row of data must contain atomic values
  • Each row of data must have a unique identifier, known as a Primary Key
- Just like First Normal Form, there is also a Second and Third Normal form. Each of those adhere to increasingly strict sets of rules.
What is a Primary Key
- The column that will be your Primary Key has to be designated as such when you create the table.
- A Primary Key is a column in your table that makes each record unique. Thus the primary key can be used to uniquely identify each record. Which, in other words, means that the data in the Primary Key CANNOT be repeated. So, for example, you would choose something like the SSN as your primary key (since every person has a unique SSN).
- For the same reasons, a Primary Key cannot be NULL either. Because other records can also be NULL. And that is not allowed.
- A Primary Key must be given a value when the record is inserted. If you insert a record without a Primary Key, you run the risk of ending up with a NULL primary key and duplicate rows in your table. This violates 1NF.
- The Primary Key must be compact. It should contain only as much information as it requires to be u nique. Nothing extra.
- The Primary Key values cannot be changed. If you could change the value of the Primary Key, you'd risk the chance of accidentally setting it to a value you have already used.
- Given all of these restrictions, often the best Primary Key is a new Primary key. Meaning, create a new column that contains a unique number. There is a debate in the SQL world about using synthetic, or made-up primary keys (like the ID column in the below example) versus using natural keys - data that is already in the table (like a VIN number on a car or the SSN number)
How to create a Primary Key
CREATE TABLE my_contacts (
contact_id INT NOT NULL AUTO_INCREMENT,
first_name varchar(30) NOT NULL,
last_name varchar(30) NOT NULL,
email varchar(30) DEFAULT NULL,
gender char(1) DEFAULT NULL,
birthday date DEFAULT NULL,
numOfPets int NOT NULL DEFAULT 0
PRIMARY KEY (contact_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci;
- We have created a new column called contact_id that will hold an integer value that will be the primary key for our table. Each value in this column will be unique and make our table atomic. The contact_id is NOT NULL because if the primary key contains a value of NULL, you can't guarantee that it will uniquely identify each row of the table. The last line specifies that the contact_id is in fact the Primary Key of the table.
- Adding the keyword AUTO_INCREMENT to our contact_id allows our SQL SW to automatically fill that column with a value starting at 1 and incrementing in steps of 1.

Chapter 5: ALTER - Rewriting the Past
- We said earlier that you cannot create a table with the same name as an already existing table. So suppose you want to make changes to an existing tbale, like adding a new column, and you want to do it without deleting and then recreating a new table with the same data and a new column. How do you do it? You can use the ALTER keyword for this purpose.
What does the ALTER command allow you to change
The ALTER command allows you to make the following changes to your table:
  • CHANGE: both the name and datatype of an existing column*
  • MODIFY: the datatype or position of an existing column*
  • ADD: a column to your table - you pick the datatype
  • DROP: a columns from your table
The * represents that a possible loss of data may occur.

ALTER TABLE my_contacts2_withoutpkey
ADD COLUMN contact_id INT NOT NULL AUTO_INCREMENT FIRST,
ADD PRIMARY KEY (contact_id);
- This command will add a column to the table that will be the Primary Key of the column. The command will also autopopulate the column with the appropriate numbers. The FIRST keyword means that the column will be inserted into the first column position.


ALTER TABLE my_contacts2_withoutpkey
ADD COLUMN phone_number VARCHAR(10) AFTER email;
The AFTER keyword tells SQL to insert the column after the email column. The AFTER keyword is optional. If you do not use it the column is added to the end of the table. Similarly, you can also use BEFORE, LAST, SECOND, THIRD, and so on.

ALTER TABLE my_contacts2_withoutpkey
RENAME TO my_contacts_2;
This will rename the TABLE.

ALTER TABLE my_contacts_3
CHANGE COLUMN contact_id my_id INT NOT NULL AUTO_INCREMENT,
ADD PRIMARY KEY (my_id);
- We are using the CHANGE COLUMN command. This means that we have to change, both, the name and the datatype of the column. The previous name of the column was contact_id that is now being changed to my_id. At the same time, we are also setting it to be the Primary key of the table. Line 3 of the command hence contains the new name of the column that we are saving - my_id.
- Remember that when you are using CHANGE COLUMN you HAVE to change both the name and the datatype of the COLUMN. Else SQL will complain about a syntax error.

ALTER TABLE my_contacts_3
CHANGE COLUMN first_name f_name VARCHAR(100),
CHANGE COLUMN last_name l_name VARCHAR(100);
You can also change two columns in one statement. Here, the earlier name of the column id first_name that is being changed to f_name.
- Remember that if you change the datatype, you might lose data. For example, going from VARCHAR(10) to VARCHAR(1) would cause "Bonzo" to change to just "B".

ALTER TABLE my_contacts_3
MODIFY COLUMN f_name VARCHAR(120);
- This will change only the datatype of the column, and leave the name unchanged. Here f_name is the name of the column that we are modifying. Take care to ensure that the new datatype does not cause any truncation of data.
- There is no direct command to change the order of the columns in the table. But remember that if you want to fetch the data in a specific order, you can use that order in the SELECT statement that you will write and the data will automatically be returned to you in the specified order.

ALTER TABLE my_contacts_3
DROP COLUMN birthday;
- This will delete the birthday column and all of the data in it as well. So use this carefully.
You can combine all of what we have seen so far to create a SQL script. Take notice of where the commas have been included, and where they have been left off. There is a single Semicolon at the very end.

How to deal with Strings
- String functions allow you to select a specified number of characters from a column. None of these String functions change the actual data stored in the table. They simply return the altered string as a result of the query. SELECT RIGHT(email, 3) FROM my_contacts_3;


SELECT SUBSTRING_INDEX(email, '@', 1) FROM my_contacts_3;
This will select everything that comes to the left of the first @ character.


ALTER TABLE my_contacts_3
ADD COLUMN user_email VARCHAR(30);
UPDATE my_contacts_3
SET user_email = SUBSTRING_INDEX(email, '@', 1);
Notice how we are creating a new column, user_email, and then using the existing data in the email column to fill the new column (lines 3 and 4).
Chapter 7: Multi-table Database Design - Outgrowing your table
- Sometimes the best way to achieve atomicity in a table is to move the non-atomic columns in the table to new tables.
What is a Database Schema
- A representation of all the structures such as tables and columns in th database along with how they connect is known as a schema
- Consider the below table for example. The interests column is not atomic. Dividing the column into multiple columns won't work either because that will break the atomicitiy as well (multiple columns containing the same type of data). One option is to move the interests to another table. Which is what we will be doing.
- But if we want to move interests to another table, we will have to uniquely identify which interest belongs to which individual. Also this new table that we create will have to be 1NF. This is how we do it:
- Note that the new table needs to be in the 1NF. Hence each record needs to be unique. For which we have the int_id (Primary Key) in the interests table. We also want to be able to uniquely identify the individual to which the value in the interest column belongs to. This is done by using the contact_id column.
What is a Foreign Key
- The Foreign Key is a column in a table that references the Primary Key of another table.
- A Foreign Key can have a different name than the Primary Key that it comes from.
- The primary key used by a foreign key is also known as a parent key. The table where the parent key is from is known as the parent table.
- The foreign key can be used to make sure that the rows in one table have corresponding rows in another table.
- Foreign key values can be null, even though primary key values can't. A NULL foreign key means that there is no matching primary key in the parent table. But we can make sure that a foreign key contains a meaningful value, one that exists in the parent table, by using a constraint.
- Foreign keys do not have to be unique.
How do you constrain a Foreign Key
- When you constrain your foreign key, you will only be able to insert values into your foreign key that exist in the table that the key came from (the parent table). This is called referential integrity. This makes sure that if you have a row in one table with a foreign key , it must correspond to a row in another through the foreign key. If you try to delete the row in the a primary key table or to change a primary key value, you will get an error if the primary key value is a foreign key constraint in another table.
- You can delete a row that has a primary key that shows up as a foreign key in some other table. But for this, you have to remove the foreign key row first. This makes sense as well because, for example, if you are removing the row from my_contacts, you don't need to know that person's interests anymore.
- You can use a foreign key to reference a unique value in the parent table. It doesn't have to be the Primary key of the parent table. It just has to be unique.
How to create a table with a Foreign Key
ALTER TABLE my_contacts_3 MODIFY my_id INT NOT NULL PRIMARY KEY AUTO_INCREMENT;
CREATE TABLE interests (
int_id INT NOT NULL AUTO_INCREMENT PRIMARY KEY,
interest VARCHAR(50) NOT NULL,
interest_id INT NOT NULL,
CONSTRAINT my_contacts_3_contact_id_fk
FOREIGN KEY (interest_id)
REFERENCES my_contacts_3 (my_id)
);

Relationships between Tables
- There are 3 possible relationships that are possible between tables:
  • one-to-one
  • one-to-many
  • many-to-many
One-To-One Relationship
- These types of tables are not that frequently used. One-to-one means that exactly one row of a parent table is related to one row of a child table.
One-To-Many Relationship
- One-to-many relationships means that a record in table A can match many records in table B but a record in Table B can only match one record in Table A. SO post about One-To-Many Relationships
- Consider the following table relationship:
- Each contact in my_contacts can have only one prof_id. But one prof_id in the professions table can occur for multiple contacts in the my_contacts table. That is - each row in professions table can have many matching rows in the my_contacts table. But each row in the my_contacts table will have only one matching row in the professions table.
Many-To-Many Relationship
- We make the use of a junction table that which contains the primary key columns of the two tables that we are trying to relate.
- A many-to-many relationship is made up of two one-to-many relationships with a junction table in between.
- Recall that in the table we saw earlier, one person had multiple interests. Also, one interest was shared by more than on person. Hence, this led to a many-to-many relationship.

What is functional dependence between two columns
- When a column's data must change when another column's data is modified, the first column id functionally dependent on the second.
- A shorthand way to define functional dependency is as follows: T.x ->; T.y: This is read like "in the relational table called T, column y is functionally dependent on the column x". Hence the statement is read from the right to the left.
- Thus, a dependent column is one containing data that could change if another column changes. Non-dependent columns stand alone.
- A partial functional dependency means that a non-key column is dependent on some, but not all, of the columns in a composite primary key.
- A transitive functional dependency is one where changing any of the non-key columns might cause any of the other columns to change.
What is the Second Normal Form
- Check this SO link for the proper answer
- 1NF is the most basic of normal forms - each cell in a table must contain only one piece of information, and there can be no duplicate rows. 2NF and 3NF are all about being dependent on the primary key. Recall that a primary key can be made up of multiple columns.
- The data depends on the key [1NF], the whole key [2NF] and nothing but the key [3NF] (so help me Codd) - For example, consider the table below:
- The toy_id and the store_id are combined to form the composite primary key. In this case, the color column depends only on the toy_id and not the store id. Hence, it is a partial functional dependency. Similarly, store_address depends on the store_id but not the toy_id. Again a partial dependency.
- A 1NF table becomes a 2NF table when there are no partial functional dependencies in the table. That is, every column in the table depends on every column that is used to build the entire primary key in that table. This will also hold true if there is only one column that is the primary key in the table.
What is the Third Normal Form
- If we are using an artificial key in our table, then achieving 2NF is easy (because that table will not contain a composite key, which means that there is a single column that forms the primary key which in turn means that every column will depend on the entire primary key by default).
- So we need to get the table into 3NF.
- For a table to be in 3 NF, we have to ensure that there are no transitive dependencies in the table. Recall that a transitive dependency occurs when changing one non-key column causes other columns to change.
